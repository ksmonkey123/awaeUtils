1.0.0 javadoc release

  • Migrated various utilities from other projects. These include:
    • Several functional types such as Try, Result, FailableFunction0, FailableFunction1 and T2 (a 2-tuple)
    • XML processing similar to X-Path
    • IterativeRunner: A runnable that evaluates the same code block in an infinite loop that can be terminated by the code block itself or by interrupting the thread running the Runnable.
    • a Source system simplifying and unifying reading data from files, streams or a URL
    • a PriorityQueue using numeric priority parameters instead of natural ordering of the elements.
    • a wrapper class for Lazy instantiation
    • a Shutdown handler interacting with Java shutdown hooks that allows the definition of a specific shutdown sequence for shutdown hooks.

0.0.6

  • added Logic system as an extension to the Java BooleanSupplier interface. Logic allows boolean operations to be done on these provider functions to enable complex behaviour. In essence any boolean composition that could be done by first evaluating multiple BooleanSupplier instances and then applying the boolean operations can be done on the Logic instances such that the complete expression can be evaluated by evaluating a single Logic instance. Some optimisation similar to classical short circuiting is done (i.e. once the result of an expression is fully known any additional providers are not evaluated).
  • added Sequence for composing runnables into a predefined interruptible sequence that can be executed asynchronously.

0.0.5 javadoc release

  • added helper methods to MachineCoreBuilder for creation of event sequences, event funnels and arbitrary sequence event chains (event sequences for any permutation of a set of events)
  • MachineCoreBuilder can provide target states for a given origin state and event
  • added CommandProcessor as a pre-built asynchronous processor that consumes commands from a StateMachine and passes them on to a given Consumer function
  • added EventGenerator as an asynchronous way to produce and issue events in a fixed rhythm. The events may either be fixed or supplied by a Supplier function
  • StateMachine states can be stored to and loaded from StateMachine.SavedState objects
  • logging support using java.util.logging.Logger
  • the structure of a StateMachine can be extracted into raw Graphviz data
  • machine cores now support terminal states
  • terminal state checking controllable through MachineCoreBuilder
  • introduced new event handling modes for StateMachine:
    • priority mode (default) processes internal events before external
    • normal mode enqueues internal processes in the main event queue
  • improved documentation

    • use StateMachineBuilder to create state machine clusters
